## 2. MASTER CHARACTER DATABASE

### Tier System Structure

**All characters follow this 4-tier progression:**

```
TIER 1: BASE FORM (0% Power Manifestation)
- Default appearance, no effects
- Usage: Dialogue, introductions, planning
- Image ID: References maintained for tracking

TIER 2: POWERED FORM (25% Power Manifestation)
- 95% identical to base + subtle effects
- Eyes glowing (character color)
- 2-3x aura, minimal environmental effects
- Usage: Tense moments, demonstrations

TIER 3: BATTLE FORM (60% Power Manifestation)
- Facial features locked to base
- Size increase or transformation ability active
- 4-5x intense aura, heavy environmental effects
- Usage: Active combat, mid-battle sequences

TIER 4: ULTIMATE FORM (100% Power Manifestation)
- Same facial features visible (or barely visible)
- Maximum transformation/power display
- 5-6x overwhelming aura, apocalyptic effects
- Usage: RARE - Final battles only (1-2 times per arc)
```

### Universal Rules (ALL CHARACTERS)

**What Stays 100% Constant Across All Tiers:**
1. Facial structure and features
2. Hair style/color (base version)
3. Core personality essence
4. Character distinctive traits
5. Eye shape (only glow changes)

**What Changes Progressively by Tier:**

| Element | Tier 1 | Tier 2 | Tier 3 | Tier 4 |
|---------|--------|--------|--------|--------|
| Eye Glow | None | Moderate | Intense | Blazing |
| Aura Size | None | 2-3x | 4-5x | 5-6x |

| Suit Evolution | Clean | Subtle runes | Clear runes | Full armor |
| Environmental Effects | None | Subtle | Destructive | Apocalyptic |
| Power Level % | 0% | 25% | 60% | 100% |

*Variable/Scaled: Matt Cole sizes up (2.5-3x), Phong Le stays elegant (2x), others as character-appropriate

## 8. PANEL CREATION WORKFLOW

### Step-by-Step Process

1. **Determine Scene Requirements**
   - Which character(s) needed?
   - What tier of power?
   - Battle or dialogue?
   - Environmental setting?

2. **Reference Character Specification**
   - Pull from Master Database
   - Check visual consistency rules
   - Identify image reference ID
   - Note emotional tone

3. **Build Prompt**
   - Use character tier specifications
   - Apply color palette
   - Include environmental details
   - Specify art style markers

4. **Generate/Review Image**
   - Check consistency against reference image
   - Verify facial features match
   - Confirm aura/power level scaling
   - Validate color palette usage

5. **Log Panel**
   - Update character panel_usage_tracking
   - Note tier and scene context
   - Increment panel_tracking counter
   - Tag narrative arc

6. **Archive for Final**
   - Save image with panel number
   - Note any branding that needs removal
   - Prepare for final output process
